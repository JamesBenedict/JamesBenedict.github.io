Longform template

A boilerplate for longform journalism articles. Made for designers learning responsive webdesign and developers wanting a streamlined workflow.


By James Benedict
Last Updated July 23, 2016

This template is meant for people of all skill levels and aims to allow designers to spend more time designing and less time coding. Tweaks and changes are encouraged as it is only the boilerplate for you to build on. You can view some examples of stories developed with earlier versions of the template here.

Alignment

There are nine layout classes to start designing with. The css is found in alignment.css .

Article

The article tag creates the main column on the screen and is where this text is now. It takes up 50% of the screen, with a 25% margin on either side.

<body>
	<article>
	 	...your content here...
	</article>
</body>

Asides

There are three type of asides, two that go inside the article tag and one that does not. For each type, there are two sepearte classes for right and left. The two tags that go inside article are aside class="left" and aside class="left-wide". If the aside class="left" is placed outside of the article, it will float to the edge of the screen.

Wide

The wide class is usually used for media or graphics. It has a width of 75% and a margin-left of 12.5%. You can either fill a div with multiple elements you want wide, or apply it to a single element.

<article> ... </article>

<div class="wide">
	...wide text here...
	...other wide elements...
</div>

<img src="img/test.jpg" class="wide">
	...wide image here...
</div>

full

The full class is a fullscreen alignment with 100% width and auto height. If you want the navbar to disappear on scrollover surround the full elements with <div id="nav-invisible"></div> and <div id="nav-visible"></div>

	<div id="nav-invisible">

	<img src="img/test.jpg" class="full">

	<div id="nav-visible">

Make your own

We can also add classes to make our own grid. I like to stick to percentages for the width and have everything float left. You can use xs-hidden, xs-visible, sm-hidden, sm-visible, md-hidden, md-visible, lg-hidden and lg-visible to hide/show an element at certain breakpoints. The tags work for all breakpoints larger than itself - thesm-hidden will keep an element hidden at md and lg breakpoints but will not hide at xs breakpoints. The xs breakpoint is under 768px, sm breakpoints are between 768px-991px, md breakpoints are between 992px-1199px and lg breakpoint is 1200px.

<article> ... </article>
	<div class="grid-wrap>
		<img src="img/test.jpg" class="grid2">
		<figcaption class="xs-visible sm-hidden"> First image's xs caption <figcaption>

		<img src="img/test.jpg" class="grid2">
		<figcaption class="xs-hidden sm-visible"> First image's other caption <figcaption>
		<figcaption> Second image's caption <figcaption>

	</div>

Ads

There are three different types of ads, all require content.js and content.css. Ads are called content because calling them ads makes it too easy for adblock. These will likely need a lot of tweaking to work with your ad servers. These images won't load until they are reached on the page. This way you can charge per view instead of per page load.

Development

I wanted to write a little about my development process to show people getting started some tools to speed up their coding. Don't get caught up trying to match this exactly, and if something gets confusing come back to it later.

Getting Started

Included is a starter folder that has an index.html with no content and only the necessary scripts and styles. Look in the app folder for examples. All elements list in app/index.html include what styles and scripts are required by them.

Tools

Sublime Text is a text editor with too many useful features to explain here. This walkthrough will get you up and going. I recommend installing the emmet plugin as well.

Sass describes itself as css with superpowers and will make your style sheets easier to work with. It has a ton a features like variables and nesting, but if you want to use them you need to send your sass files through a preprocessor. This automaticall converts the sass file to css for the webpage. There are a few options to do this but my favorite is gulp.

gulp is a javascript library that automatically does tasks you don't want to do yourself. Want your browser to reload when you change a style? Want minifyed images? Want to convert your sass files to css? Gulp does all these and more with a single command line. This tutorial should get you up and running.

File Structure

This project is set up with gulp in mind so I have an app folder where I develop and and dist folder to publish. The gulp tutorial above will have automate certain tasks like image and javascript minification, which are then copied into the dist folder. Performace is a feature so don't overlook these efficiency improvements. If you don't use gulp do these tasks by hand before publishing.

I like to break up my styles and scripts into smaller files instead of having one master style.css or script.js. This helps me find what I'm looking for faster. Gulp will concatenate all these files into one for the dist version. Fewer requests will make your page load faster.
